Guides and Tutorials for Learning PHP

Note: This guide assumes you already have an understanding of HTML/CSS. It is highly recommended that you understand HTML/CSS before learning PHP. Freecodecamp offers practical tutorials in learning HTML/CSS/JS.

New to Programming, New to PHP

You're new to the world of programming and wish to cut your teeth with PHP. You already have the fundamentals of HTML/CSS.

  • https://github.com/PeeHaa/php-net-tutorial - An updated PHP tutorial guide that assumes no prior programming experience

Experienced Programmer, New to PHP

You're already familiar with programming, but want to add PHP to your skillset.

  • https://secure.php.net/manual/en/language.variables.basics.php - PHP variable basics
  • https://secure.php.net/manual/en/language.types.intro.php - Introduction to PHP data types
  • https://secure.php.net/manual/en/control-structures.intro.php - Introduction to PHP control structures
  • https://secure.php.net/manual/en/language.functions.php - Functions in PHP, pay the most attention to User-defined functions, Returning values, Variable functions is useful to know, but may not be necessary to learn at this point, Anonymous functions are PHP's implementation of closures.
  • https://secure.php.net/manual/en/language.oop5.basic.php - the basics to object-oriented PHP
  • https://secure.php.net/manual/en/reserved.variables.php - a list of pre-defined variables in PHP
  • https://secure.php.net/manual/en/language.oop5.php - and if desired, here's the table of contents from the PHP's manual on object-oriented PHP
  • https://secure.php.net/manual/en/language.namespaces.rationale.php - namespaces in PHP

Intermediate

After getting the bssics under your belt, here are some other resources to help tie it together.

  • https://www.phptherightway.com/
  • https://github.com/PatrickLouys/no-framework-tutorial (Note: This is slightly outdated in some parts, but it is still a good, practical tutorial in learning how to write professional PHP)

Security

Keeping your application secure is important! To quote OWASP's Top 10 2017: the harm [of a vulnerable application] that is caused may be of no consequence, or it may put you out of business. Ignoring your application's security is not a valid defense in protecting yourself.

  • https://paragonie.com/blog/2017/12/2018-guide-building-secure-php-software - a newer guide on writing secure PHP software
  • http://php.net/manual/en/faq.passwords.php - explanation on password hashing

Database Interaction

There will likely become a time that you'll need to interact with a database. PDO is best suited to this task.

  • http://wiki.hashphp.org/PDO_Tutorial_for_MySQL_Developers - learning how to use PDO when coming from a MySQL/MySQLi background
  • https://phpdelusions.net/pdo - alternatively, this is a thorough guide on learning PDO
  • https://stackoverflow.com/questions/134099/are-pdo-prepared-statements-sufficient-to-prevent-sql-injection - using prepared statements in PDO is of the utmost importance, but they're not completely infallible, this explains how to further protect your queries while using prepared statements

Advanced

Alright, you've made it this far, and the quirks of PHP haven't scared you away. Here's additional material to further your learning.

  • https://patricklouys.com/professional-php/ This is a book written by the same author for the No-Framework Tutorial, but is updated.
  • https://stackoverflow.com/questions/5863870/how-should-a-model-be-structured-in-mvc - an explanation of what models are in the MVC relationship
  • https://stackoverflow.com/questions/16356420/mvc-for-advanced-php-developers - a list of videos and reading material

Development Environment

PHP isn't the only thing you should learn! A useful skill is knowing how to set up your own development environment without using XAMPP. Below are guides on setting up different development environments. (Note that VM means Virtual Machine)

Vagrant

https://www.vagrantup.com/intro/getting-started/index.html - Vagrant is a software that allows you to set up a development environment VM quickly. You can find vagrantboxes that other people have created and uploaded, in which you can use instead of setting up a VM in Vagrant from scratch

Docker

https://docs.docker.com/get-started/ - another common development environment is Docker. It is an alternative to VMs because it does not require the setting up and maintenance that regular VMs do. It is able to do this through the use of containers.

Rolling Your Own VM

The options below are guides on setting up a development environment from scratch using a VM on your computer.

  • Setting up a LAMP environment (Linux [Ubuntu], Apache, MySQL, PHP)
  • Setting up a LEMP environment (Linux [Ubuntu], nginx, MySQL, PHP)
  • Another setting up a LEMP environment (Linux [Arch], nginx, MySQL, PHP

Linux

  • https://www.digitalocean.com/community/tutorials/an-introduction-to-linux-basics - Linux basics

In regards to troubleshooting in Linux, Google is your best friend. Seriously. If you have an error or problem, google the error or the nature of the problem, 90% of the time, someone else has encountered your situation as well, and you can find an answer pretty quickly. 10% of the time, it's a weird issue, and you'll either need to do more digging through google with some clever keywords, or ask someone.

Some other notes...

If you have an aversion to Oracle products, you can use mariaDB instead of MySQL which is virtually the same as MySQL, except that it is not owned by Oracle.

Lastly, this guide explains how to set up shared folders in Virtualbox from Windows. It took me a lot of digging to find it, and it was extremely helpful for me.

Book Recommendations

Most of these are not specific to PHP, but if you're serious and dedicated to learning how to develop, here are some books that will help you. I've chosen not to provide links to these books, as your preferred method of purchasing books may be different than mine. Googling each book will give you options on how to purchase them.

  • Clean Code by Robert Martin
  • Refactoring by Martin Fowler and Kent Beck
  • Pragmatic Programmer by Andy Hunt and Dave Thomas
  • Patterns of Enterprise Application Architecture by Martin Folwer, Dave Rice
  • Working Effectively with Legacy Code by Michael Feathers (if you're ever dumped onto a project that's using an unsupported version of PHP...)
  • Modern PHP by Josh Lockhart - created by the author of https://www.phptherightway.com/
